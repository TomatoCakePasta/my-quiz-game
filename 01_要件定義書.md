# 01_要件定義書

## 1. プロジェクト概要
本プロジェクトは、ゲーミフィケーションを活用して知識のアウトプットと定着を促進するクイズアプリを開発する。

## 2. 目的
- プレイヤーの自主学習を支援し、学習内容の定着を高める
- クイズ形式で知識をアウトプットする場を提供する
- ゲーム要素によるモチベーション向上と継続利用の促進

## 使用技術
- フロントエンド: Vue.js, Vuetify
- バックエンド: FastAPI (REST API)
- データベース: PostgreSQL
- 実行環境: Docker を利用したコンテナ構成
- 通信: WebSocket を利用したリアルタイム機能
- アーキテクチャ: MVCモデル

## 3. 主な機能

### 3.1 プレイヤー機能
- 自主練習
  - クイズに挑戦し、正答率やスコアを確認できる
  - 練習履歴や学習記録を蓄積できる
- 問題の投稿
  - 管理者がジャンルを指定して問題募集を行う
  - プレイヤーは問題募集リストから応募できる
  - 応募後は管理者の承認待ちとなる
- 大会の参加
  - 開催中の大会に参加してランキングを競う
  - 大会の成績や順位を確認できる
- ランク確認
  - プレイヤーの現在ランクやランク推移を確認できる
  - 獲得ポイントや実績を表示する

### 3.2 管理者機能
- 問題の募集
  - 管理者がジャンルを指定して問題募集を行える
  - 募集期間や募集対象を設定できる
- 投稿問題の承認可否
  - プレイヤーが応募した問題を一覧で確認できる
  - 承認・却下を行い、公開可否を管理できる
- 大会の運営
  - 大会の作成、開催期間設定、参加条件設定を行う
  - 大会結果の集計と順位付けを行う
  - 大会の進行状況を管理できる

## 4. 利用対象者
- 知識をアウトプットして定着させたい個人学習者
- クイズを通じて学習を継続したいプレイヤー
- コンテンツ作成や学習の場を管理したい運営者・管理者

## 5. 非機能要件
- モバイルおよびデスクトップ環境から利用しやすいUI
- 問題データとプレイヤーデータの安全な管理
- 大会や投稿問題の管理機能の操作性
