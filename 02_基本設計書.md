# 02_基本設計書

## 1. 概要
本基本設計書は、`01_要件定義書.md` に基づき、クイズアプリのシステム構成、画面構成、データ構造、機能詳細を整理する。

## 2. システム構成
- Webアプリケーションまたはシングルページアプリとして開発
- プレイヤーと管理者の2つのロールを持つ
- クイズ問題、プレイヤー情報、大会情報を管理するデータストアを想定
- UIはモバイルとデスクトップの両方で使いやすい設計にする

## 使用技術
- フロントエンド: Vue.js, Vuetify (コンポーネントベースでUIを構築)
- バックエンド: FastAPI を用いた REST API
- データベース: PostgreSQL
- 通信: WebSocket によるリアルタイム通知・大会進行
- アーキテクチャ: MVCモデルに沿った責務分離

## 3. 画面一覧

### 3.1 プレイヤー画面
- ログイン / サインアップ
- ダッシュボード
  - 今日の練習状況、獲得ポイント、ランク
- 自主練習画面
  - クイズに挑戦、問題の解答、正答率表示
- 問題投稿画面
  - 問題文、選択肢、解答、カテゴリ、難易度を入力
- 大会一覧 / 参加画面
  - 開催中および近日開催の大会を表示
- ランク確認画面
  - ランク、ポイント、実績、推移を確認

### 3.2 管理者画面
- 管理者ログイン
- 問題管理画面
  - 問題作成、編集、公開状況管理
- 問題募集管理画面
  - 管理者がジャンルを指定して問題募集を作成・管理
  - 募集状況や応募状況を確認
- 投稿問題承認画面
  - プレイヤー応募問題の一覧、承認・却下操作
- 大会管理画面
  - 大会作成、開催期間設定、参加条件設定、進行状況管理
- 結果確認画面
  - 大会結果の集計と順位表示

## 4. データ構造

### 4.1 プレイヤー
- ID
- 名前 / ユーザー名
- ロール（プレイヤー / 管理者）
- ランク
- ポイント
- 実績
- 練習履歴

### 4.2 クイズ問題
- ID
- 作成者ID
- 問題文
- 選択肢
- 正解
- カテゴリ
- 難易度
- 公開状態（公開 / 承認待ち / 却下）
- 作成日 / 更新日

### 4.3 大会
- ID
- タイトル
- 説明
- 開催開始日時 / 終了日時
- 参加条件
- ステータス（開催中 / 終了 / 予定）
- 参加者リスト
- 結果・順位

### 4.4 練習記録
- ID
- プレイヤーID
- 問題ID
- 解答結果
- スコア
- 実施日時

## 5. 機能詳細

### 5.1 プレイヤー機能
- 自主練習
  - 問題をランダムまたはカテゴリごとに出題
  - 解答後に正否と解説を表示
  - 練習記録を保存して履歴を参照
- 問題の投稿
  - 管理者が指定したジャンルの問題募集リストから応募できる
  - 応募問題は管理者承認待ち状態になる
- 大会の参加
  - 大会一覧から参加可能な大会を表示
  - 大会中に指定問題を解き、成績を送信
- ランク確認
  - 獲得ポイントや実績に基づいてランク表示
  - 最新のランクと推移をグラフまたは一覧で提示

### 5.2 管理者機能
- 問題の募集
  - 管理者専用の問題募集作成画面を用意
  - ジャンルを指定して募集を開始し、募集期間や対象を設定
- 投稿問題の承認可否
  - プレイヤーが応募した問題の一覧画面から承認／却下を実行
  - 承認後は公開状態に変更し、プレイヤーが利用可能にする
- 大会の運営
  - 大会作成時に期間と参加条件を設定
  - 参加者の進捗や成績を集計
  - 大会終了後に順位を確定・公開

## 6. 画面遷移
- ログイン → ダッシュボード
- ダッシュボード → 自主練習／問題投稿／大会一覧／ランク確認
- 管理者ダッシュボード → 問題管理／投稿問題承認／大会管理

## 7. 非機能設計
- ユーザー操作のレスポンスを速くする
- データは安全に保存・取り扱う
- UIはシンプルで直感的に操作できること
- モバイルファーストのレイアウトを意識する

## 8. 追加留意点
- 01の要件に沿って、ユーザーと管理者の権限を明確に分ける
- 投稿問題の承認フローを実装可能な形で設計する
- 大会運営機能は開始／終了と成績集計を中心に設計する
