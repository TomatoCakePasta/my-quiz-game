# ===== 構成 =====
- frontend: Vue.js + Vuetify + Vite
- backend: FastAPI + WebSocket
- DB: PostgreSQL
- UI Test: Playwright
- Unit Test: pytest
- Other: Docker

CICDにGitHub Actionsを使いたいが、  
今回はこの構成でキャパオーバーのため次回課題とする

# ===== Docker起動法 =====
```
# プロジェクトディレクトリで
docker compose up

# これだけで3つコンテナ立ち上がって、アクセスできる
- http://localhost:3000 -> Vue.js
- http://localhost:8000 -> FastAPI
- http://localhost:5050 -> pgAdmin
```

# 開発サイクル
```
# 初回起動
docker compose up --build
```

以下最初にやったプロジェクト作成
# ===== 1. フロントエンド =====
```
# プロジェクトディレクトリで実行
# (vuetifyのプロジェクト名をfrontendにすると、/プロジェクトディレクトリ/frontendになる)
npm create vuetify@latest
```

# ===== 2. バックエンド =====
- 普通にbackendフォルダを作成

```
# backend/requirements.txtを作成 (インストールしたいツールを記入)
fastapi
uvicorn[standard]
```

```
# backend/main.pyを作成
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def hello():
    return {"message": "hello"}

```

- インストール
```
# 仮想環境の作成 (新しいターミナルを開くたびに行う)
python -m venv venv

# 仮想環境の軌道
source venc/bin/activate

# 必要なツールのインストール(プロジェクト作成時のみ)
pip install -r requirements.text
```
# ===== 3. Docker化 =====
- frontend, backendでそれぞれDockerfileを記述
- docker-compose.ymlで起動設定を記述
　(postgres, pgadminも)
- dockerignoreで無視するファイルを記述
- 以下のコマンドで一旦起動
```
docker compose up --build

# アクセス確認
- http://localhost:3000 -> Vue.js
- http://localhost:8000 -> FastAPI
- http://localhost:5050 -> pgAdmin (username: admin@example.com, pass: admin)

```

pytest, playwrightの追加
- backend側: requirements.txtに以下のライブラリを追記
```
# test run library
pytest
# async function test
pytest-asyncio
# FastAPI API test
httpx
```

- frontend側
```
npm install -D @playwright/test

# 初回 setup
npx playwright install

# generate tests folder
# 設定は全てY (True)
# GitHub Actionsとかも追加してくれた (playwright.yml)
npm init playwright@latest

# テスト実行
npx playwright test
```
