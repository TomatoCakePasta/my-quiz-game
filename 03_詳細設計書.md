# 03_詳細設計書

## 1. 目的
`01_要件定義書.md` と `02_基本設計書.md` に基づき、本詳細設計書ではシステムの具体的な構成、画面仕様、API仕様、データモデル、フローを段階的に整理する。

## 2. システム構成詳細
- フロントエンド: Vue.js + Vuetify
- バックエンド: FastAPI
- データベース: PostgreSQL
- 実行環境: Docker を利用したコンテナ構成
- 通信: REST API、WebSocket
- テスト: フロントエンドは Playwright による E2E テスト、バックエンドは pytest + TestClient による API テスト
- アーキテクチャ: MVCモデル
  - Model: データモデルとドメインロジック
  - View: Vue/Vuetify による画面表示
  - Controller: FastAPI のエンドポイント、ビジネスロジック制御

## 3. 画面詳細

### 3.1 プレイヤー画面
#### 3.1.1 ダッシュボード
- 表示項目
  - 今日の練習達成度
  - 獲得ポイント
  - 現在ランク
  - 進行中・近日開催の大会
- 遷移先
  - 自主練習画面
  - 問題応募一覧画面
  - 大会一覧画面
  - ランク詳細画面

#### 3.1.2 自主練習画面
- 表示項目
  - 出題される問題文
  - 選択肢（4択想定）
  - 解答送信ボタン
  - 正答率・経過時間
- 振る舞い
  - 問題を取得し、回答後に結果と解説を表示
  - 再挑戦または次の問題へ遷移

#### 3.1.3 問題応募一覧画面
- 表示項目
  - 管理者が募集しているジャンルと募集概要
  - 募集期限
  - 応募ボタン
- 振る舞い
  - 応募ボタンで問題応募フォームに遷移
  - 募集が終了している場合は応募不可表示

#### 3.1.4 問題応募フォーム
- 入力項目
  - 問題文
  - 選択肢1〜4
  - 正解番号
  - カテゴリ/ジャンル（管理者募集に紐づくジャンルが自動設定される）
  - 難易度
- 振る舞い
  - 入力バリデーション: 必須チェック、選択肢重複禁止
  - 送信後は「承認待ち」状態として保存

#### 3.1.5 大会一覧画面
- 表示項目
  - 大会タイトル
  - 開催期間
  - 参加条件
  - ステータス（予定/開催中/終了）
- 振る舞い
  - 開催中大会は参加ボタン表示
  - 参加済み大会は進捗リンクを表示

#### 3.1.6 ランク確認画面
- 表示項目
  - 現在ランク名
  - ランクポイント
  - 過去のランク推移グラフ
  - 実績バッジ一覧

### 3.2 管理者画面
#### 3.2.1 問題募集管理画面
- 表示項目
  - 募集タイトル
  - ジャンル
  - 募集期間
  - 応募数
  - ステータス（募集中/終了）
- 操作
  - 募集作成
  - 募集編集
  - 募集終了

#### 3.2.2 投稿問題承認画面
- 表示項目
  - 応募者名
  - 問題文
  - 選択肢
  - 正解
  - 応募ジャンル
  - 応募日時
- 操作
  - 承認
  - 却下
  - コメント記入

#### 3.2.3 大会管理画面
- 表示項目
  - 大会タイトル
  - 開催期間
  - 参加条件
  - 参加者数
  - 結果ステータス
- 操作
  - 大会作成
  - 期間変更

## 4. API仕様

### 4.1 認証
- POST /auth/login
  - request: { email, password }
  - response: { access_token, token_type, role }
- POST /auth/register
  - request: { name, email, password }
  - response: { user_id, role }

### 4.2 プレイヤー関連
- GET /players/{player_id}
  - プレイヤー情報取得
- GET /players/{player_id}/dashboard
  - ダッシュボードデータ取得
- GET /players/{player_id}/rank
  - ランク情報取得

### 4.3 問題募集・応募関連
- GET /recruitments
  - 募集中の問題募集リスト取得
- GET /recruitments/{id}
  - 募集詳細取得
- POST /recruitments/{id}/applications
  - 問題応募送信
  - request: { question_text, options, answer_index, difficulty }
- GET /applications/{application_id}
  - 応募詳細取得

### 4.4 管理者問題管理関連
- POST /recruitments
  - 問題募集作成
  - request: { title, genre, description, start_at, end_at }
- PUT /recruitments/{id}
  - 募集編集
- GET /applications
  - 全応募一覧取得
- PUT /applications/{id}/approve
  - 応募承認
- PUT /applications/{id}/reject
  - 応募却下

### 4.5 大会関連
- GET /tournaments
  - 大会一覧取得
- POST /tournaments
  - 大会作成
  - request: { title, description, start_at, end_at, entry_criteria }
- PUT /tournaments/{id}/start
  - 大会開始
- PUT /tournaments/{id}/finish
  - 大会終了・結果集計
- GET /tournaments/{id}/results
  - 結果取得

### 4.6 WebSocket
- エンドポイント: /ws
- 送信イベント
  - `tournament_update`: 大会進行状況の更新
  - `application_status`: 応募承認/却下通知
- 受信イベント
  - `subscribe`: クライアントがトピックを購読

## 5. データモデル詳細

### 5.1 プレイヤー
- id: UUID
- name: string
- email: string
- password_hash: string
- role: enum(player, admin)
- rank: string
- points: integer
- achievements: list[string]
- created_at: datetime
- updated_at: datetime

### 5.2 募集
- id: UUID
- title: string
- genre: string
- description: string
- start_at: datetime
- end_at: datetime
- status: enum(open, closed)
- created_by: UUID (管理者ID)
- created_at: datetime
- updated_at: datetime

### 5.3 応募問題
- id: UUID
- recruitment_id: UUID
- player_id: UUID
- question_text: string
- options: list[string]
- answer_index: integer
- difficulty: enum(easy, medium, hard)
- status: enum(pending, approved, rejected)
- submitted_at: datetime
- reviewed_at: datetime | null
- reviewer_id: UUID | null

### 5.4 大会
- id: UUID
- title: string
- description: string
- start_at: datetime
- end_at: datetime
- entry_criteria: string
- status: enum(upcoming, active, completed)
- participants: list[UUID]
- results: list[object]
- created_at: datetime
- updated_at: datetime

### 5.5 練習記録
- id: UUID
- player_id: UUID
- question_id: UUID
- is_correct: boolean
- score: integer
- attempted_at: datetime

## 6. 検証に向けた段階的開発計画

### 6.1 第1段階: 基本機能の実装
- フロントエンド: ダッシュボード、募集一覧、応募フォーム、管理者募集管理画面
- バックエンド: 認証、募集一覧、応募登録、募集作成、応募一覧、承認/却下
- データ: プレイヤー、募集、応募問題を保存
- 検証
  - 募集一覧が表示される
  - 応募フォームから応募できる
  - 管理者が募集を作成できる
  - 管理者が応募を承認/却下できる

### 6.2 第2段階: 自主練習とランク機能
- フロントエンド: 自主練習画面、ランク確認画面
- バックエンド: 問題出題、解答受付、練習記録保存、ランク計算
- 検証
  - 自主練習が問題を提示する
  - 解答後に結果が返る
  - 練習記録が保存される
  - ランクが反映される

### 6.3 第3段階: 大会運営とリアルタイム通知
- フロントエンド: 大会参加、結果表示、WebSocket通知
- バックエンド: 大会作成・進行・結果集計、WebSocketイベント配信
- 検証
  - 大会作成・参加ができる
  - 大会進行ステータスが更新される
  - WebSocket で通知が届く

### 6.4 第4段階: テストと運用準備
- テスト: 正常系・異常系のエンドツーエンド検証
- ドキュメント: API 仕様、画面遷移、データモデルの確認
- 運用: セキュリティ、スケーラビリティ、ログ管理

## 7. 注意点
- 管理者・プレイヤーの権限分離を厳密に行う
- 問題応募と承認のフローは必ずステータス遷移を管理する
- 大会中の成績はリアルタイムに更新し、結果公開まで整合性を保つ
