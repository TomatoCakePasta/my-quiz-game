# 04_テスト仕様書

## 1. 目的
`03_詳細設計書.md` の設計に基づき、クイズアプリの正常系・異常系のテスト項目と手順を記載する。アプリの主要機能とAPIの動作を検証し、設計どおりに実装されているかを確認する。

## 2. テスト対象
- 認証機能
- プレイヤーのダッシュボード表示
- 問題募集一覧・応募機能
- 応募問題の承認/却下フロー
- 大会一覧・参加機能
- WebSocket通知
- データ整合性と権限管理

## 3. テスト環境
- フロントエンド: Vue.js + Vuetify
- バックエンド: FastAPI
- データベース: PostgreSQL
- 実行環境: Docker を利用したコンテナ構成
- ブラウザ: 最新の Chrome または Firefox
- テストツール
  - フロントエンド: Playwright による E2E テスト
  - バックエンド: pytest + FastAPI TestClient による API テスト
  - API 単体: Postman / REST API クライアント

## 6. テスト観点
- 正常系/異常系ともに API レスポンスコードとレスポンスボディを確認する
- 権限チェックが想定どおりに機能していること
- データの状態遷移（応募: pending→approved/rejected、大会: upcoming→active→completed）が正しく行われること
- WebSocket イベントが適切なタイミングで配信されること

## 7. テスト結果記録
- テスト項目ごとに実施日、実施者、結果（合格/不合格）、コメントを記録する
- 不合格の場合は再現手順と修正予定を明確にする
